# K1：A2 广域 LoRA 上的完整 SNSCL 核心机制

日期：2026-07-20

## 研究判断

H1 只在缓存 CLIP 特征上加入高斯扰动并做 batch 内对比，缺少论文 SNSCL 的三个关键组成，因此其失败不能证伪 SNSCL。CVPR 2023 原论文面向的正是细粒度噪声标签分类，并指出普通 SCL 在该场景可能下降；有效版本同时包含：不可信锚点的加权标签修正、按可信度选择性更新的逐类 FIFO 队列、由均值和方差网络产生的随机特征及 KL 防塌缩项。

K1 在 J0 的正向广域 LoRA 基线上加入上述训练期辅助机制。A2 的 cross-fitted trust 取代论文的训练内 loss-GMM 可靠度，以避免同一模型对自身损失进行循环确认。最终推理丢弃随机模块、投影器、队列和标签记忆，仍是单一 OpenAI CLIP ViT-B/32、单个线性头和一组 LoRA 权重。

## 冻结实验设计

- 控制：已完成的 J0；K1 与 J0 使用同一 A2 epoch48 初始检查点、同一 90,204 张 A2 保留集、同一未见验证集、相同 seed42、2 epoch、batch64、weak RRC + flip、GCE q=0.5、连续 trust 加权、feature distillation 2.0、head LR 5e-5、LoRA LR 2e-5、末四层 rank8 Q/V/out LoRA。
- 随机模块：512→2048→两路 512 维均值/对数方差，均值为 identity-preserving residual；重参数采样后经单层 128 维投影器。
- 队列：500 个类别各自长度 32 的 FIFO；可靠度高于 0.5 时必定入队，否则以可靠度为概率入队；对比温度 0.07。
- 锚点标签：`omega=1 if trust>0.5 else trust`，`soft=(1-omega)*model_prediction+omega*noisy_onehot`；首次观察直接初始化，后续以 0.99 EMA 平滑，硬标签仅用于逐类对比队列。
- 损失：J0 原损失 + `0.05*L_NTCL + 0.001*L_KL`。J0 分类损失约 0.38，而满队列 NTCL 数量级约为 8--10，0.05 用于让新增表示梯度与主任务同阶，避免训练期辅助目标淹没已有强检查点。
- A2 保留集 trust 均值 0.7644；在阈值 0.5 下每 epoch 预计约 71,849 次入队，远高于队列总容量 16,000，足以覆盖全部类别并持续刷新。

## 预注册门槛

1. K1 clean-core micro 相对 J0 至少 `+0.50pp`；
2. K1 trusted macro 不低于 J0，raw micro 不得下降超过 `0.20pp`；
3. 第二 epoch 的可用对比锚点比例至少 95%，500×32 队列必须全部填满；
4. 若第 1 条失败，不追加 epoch、不扫描 loss weight/temperature/queue size、不生成平台提交包；
5. 只有同时通过前三条，才允许进入裸推理与 flip-TTA 单模型提交审计。

## 合规与隔离

只读取官方训练数据、A2 已有 cross-fitted 训练证据和未见验证集；测试集仅保留给最终推理。队列是单个模型训练过程的记忆库，不是多个模型的融合或投票；随机模块与投影器不会出现在推理路径。团队仓库保持只读，所有代码、日志和输出均位于独立工作树 `AegisCLIP-F6-A2LoRA`。

## 严格结果

最终有效运行使用单一主进程从干净输出目录完成。此前两个启动尝试分别因外层 120 秒时限及其遗留 WSL 子进程而作废；发现并发后已按精确 K1 命令行全部终止，随后 `--overwrite` 清空输出重新运行，作废文件与有效证据不存在混用。

epoch1 已填满 500×32 队列，入队 71,931 次、可用锚点 99.2495%；epoch2 入队 71,895 次、可用锚点 100%，`L_NTCL` 从 7.7543 降到 6.5130，证明新增机制实际执行并收敛。最终与 J0 最佳结果比较：

- clean-core micro：K1 `82.8389227%`，J0 `82.8389227%`，差值 `+0.0000pp`；
- clean-core macro：K1 `82.7496231%`，J0 `82.7478766%`，差值 `+0.0017pp`；
- raw micro：K1 `69.7248578%`，J0 `69.7442353%`，差值 `-0.0194pp`；
- trusted macro：K1 `80.6933224%`，J0 `80.7065427%`，差值 `-0.0132pp`；
- flip agreement：K1 `88.1321430%`，J0 `88.0740166%`，差值 `+0.0581pp`。

主门槛要求 clean-core 至少 `+0.50pp`，实际为零，因此 K1 失败。按预注册不追加 epoch、不扫描超参数、不生成平台提交包。

## 失败归因

训练辅助状态显示 90,204 张样本均完成标签记忆初始化，其中 9,883 张（10.9563%，源类覆盖 500/500、目标覆盖 499/500）的队列硬标签偏离原标签；这些改动样本在原类上的平均软质量仅 0.0953。相对 J0，K1 的 24 个 LoRA 张量相对 L2 变化约 2.775%，分类头也发生非零变化，因此不是“辅助损失没有梯度”。然而 clean-core 正确数完全不变且 raw/trusted 略降，说明模型优化了由自身预测产生的队列目标，却没有获得新的外部判别信息，属于同源自确认而非有效去噪。
