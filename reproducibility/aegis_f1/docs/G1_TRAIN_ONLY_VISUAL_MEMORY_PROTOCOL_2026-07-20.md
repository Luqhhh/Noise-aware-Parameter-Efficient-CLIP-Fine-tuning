# G1：纯训练集视觉记忆结构诊断

日期：2026-07-20

## 要回答的问题

当前 A2 只用一个 512→500 的线性头，把同一物种在性别、季节、姿态、背景和尺度上的
多模态外观压到一个超平面决策中。G1 检验：官方 CLIP 特征中是否仍保存着线性头没有利用的
局部邻域信息。若答案为是，后续应蒸馏视觉记忆结构，而不是继续微调学习率。

## 冻结协议

- 记忆库：A2 strict-train，物理删除 991 个 A2 高置信 reject；
- 查询集：F6 的 A2-held-out evaluation；任何与记忆库共享内容 SHA-256 组的查询都剔除；
- 特征：同一个官方 OpenAI CLIP ViT-B/32 冻结特征缓存；
- 固定 `k=10`、`beta=20`，不扫描 k、温度或混合系数；
- 两个具有因果含义的库：`a2_kept` 与其中 `OOF clean_probability>=0.70` 的子集；
- 对照：平台最优 A2 epoch 48 线性头；
- 不读取测试图像、测试预测或测试分布，不做测试时适应。

## 预注册判据

本轮只判断结构机会，不直接产生提交：

1. 若任一固定记忆头的 clean-core micro 比 A2 高至少 1.0pp，则进入可部署单头设计；
2. 或者，若 A2 与记忆头的 clean-core oracle 至少比 A2 高 5.0pp，说明存在可学习的互补结构，
   进入严格嵌套拆分的门控/蒸馏实验；
3. 两项均不满足，则淘汰样本级视觉记忆主线，不通过调 k 或 beta 挽救；
4. 即使通过，也不得直接将两个输出简单融合后上传。发布形态必须是一个可审计的单模型推理流程，
   且需再次核对赛事对“模型融合”的解释。

## 成功量级

61.21→90 的差距决定了 G1 不能以 0.1–0.5pp 为成功。1pp 只够证明方向有信号；真正推进到
平台的候选必须在隔离验证上展示数个百分点的可复现潜力，或者给出同量级的互补上限。

## 固定运行结果

内容隔离后查询 9,267 张，另剔除 48 张与 A2 train 共享内容组的图像；clean-core 为 6,440 张。

| 预测器 | raw micro | clean-core micro | 与 A2 的 clean-core oracle |
|---|---:|---:|---:|
| A2 epoch 48 线性头 | 69.4723% | 83.0590% | 83.0590% |
| A2-kept 记忆库 | 64.7675% | 76.1646% | 87.1739% |
| OOF-clean 记忆库 | 59.6633% | 76.2422% | 87.3292% |

最佳 oracle 增量为 **+4.2702pp**，低于预注册的 +5.0pp 结构门槛；记忆头单独也比
A2 低 6.8168pp。因此 G1 判负并终止，不扫描 `k`、`beta` 或融合系数，不生成提交包。

## 解释

冻结 CLIP 邻域确实能挽救 275 个 A2 clean-core 错例，但同时破坏 714 个 A2 正确样本。
这说明局部外观信息存在但不够纯，样本级 kNN 不是 61→90 的决定性遗漏。下一主线应处理
更大范围的标签结构和干净、类别均衡测试分布，而不是把两个较弱输出拼接。
