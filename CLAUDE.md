# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

Fine-grained image classification (500 classes, ~103K training images) with noisy labels, using CLIP ViT-B/32 as the backbone. This is a competition entry — the preliminary round requires a single-model baseline with no ensembling or test-time training.

## Architecture

```
common/          → Shared code: dataset classes, config/seed/logging utils, submission generator
experiments/     → One subdirectory per method (baseline/, lora/, prototype/, etc.)
configs/         → One YAML per experiment, pointing to the corresponding experiment and output dirs
scripts/         → Data preparation and submission validation (experiment-agnostic)
```

**Key principle:** `common/` contains everything that doesn't change between experiments. Each experiment under `experiments/<name>/` is self-contained (model.py, train.py, evaluate.py, infer.py) and imports from `common`. Outputs go to `outputs/<experiment_name>/` to avoid cross-experiment contamination.

## Common Commands

All commands run from repo root.

```bash
# Data inspection
python3 scripts/check_data.py --config configs/baseline.yaml

# Train/val split (required before first training run)
python3 scripts/split_data.py --config configs/baseline.yaml

# Training (resume with --resume outputs/baseline/checkpoints/last.pt)
python3 -m experiments.baseline.train --config configs/baseline.yaml

# Evaluate on validation set
python3 -m experiments.baseline.evaluate --config configs/baseline.yaml --ckpt outputs/baseline/checkpoints/best.pt

# Test-set inference → produces outputs/baseline/submissions/pred_raw.csv
python3 -m experiments.baseline.infer --config configs/baseline.yaml --ckpt outputs/baseline/checkpoints/best.pt

# Generate submission files (pred_results.csv + submission.zip)
python3 -m common.submission --raw outputs/baseline/submissions/pred_raw.csv --out_dir outputs/baseline/submissions

# Validate submission before uploading
python3 scripts/check_submission.py --test_dir /path/to/test --csv outputs/baseline/submissions/pred_results.csv --zip outputs/baseline/submissions/submission.zip
```

## Adding a New Experiment

1. Create `experiments/<name>/` with `__init__.py`, `model.py`, `train.py`, `evaluate.py`, `infer.py`
2. Create `configs/<name>.yaml` pointing `split_dir`, `save_dir`, `log_dir`, `submission_dir` to `outputs/<name>/...`
3. Import from `common.dataset`, `common.utils`, `common.submission` — do not copy these files
4. Run: `python3 -m experiments.<name>.train --config configs/<name>.yaml`

## Key Design Details

**CLIP dtype handling:** CLIP ViT-B/32 loads with `conv1` in fp16 on CUDA but other parameters in fp32. `build_model()` converts the visual encoder to float32 via `.float()` to avoid dtype mismatches. Without this, forward passes fail with "Input type and weight type should be the same".

**Class mapping:** Classes are mapped by sorting directory names lexicographically → indices 0..499. `class_to_idx.json` and `idx_to_class.json` are generated by `split_data.py` and reused by training/inference. Never hardcode class indices.

**Submission format:** `pred_results.csv` must have format `image_name.jpg, 0001` (comma + space, 4-digit zero-padded label). The check script validates 9 criteria: filename, line count, field count, image name existence, duplicates, coverage, label format, label range, zip contents.

**Data split convention (2026-07-17):** All new experiments MUST use `outputs/d3_strict/seed42` as `split_dir`. This is the group-aware split that prevents cross-class duplicate leakage by enforcing same-SHA-256 images to stay in the same fold. The older `outputs/ref/seed42` split is deprecated — do not use it for new configs. All OOF weight manifests are tied to d3_strict, and using ref would cause manifest key mismatches.

**Sample weight interface:** `TrainImageDataset` returns `(image, label, image_path)` and has `get_sample_weights()` method. This is intentional — future noise-robust methods can use `image_path` to assign per-sample weights without changing the dataset code.

**Corrupted image handling:** `_safe_load_image()` returns `None` on failure; callers substitute a zero tensor so training doesn't crash. PIL's `LOAD_TRUNCATED_IMAGES = True` is set at module level.

## Config System

YAML files loaded by `common/utils.py:load_config()`. Key sections:

- `data`: train_dir, test_dir, split_dir, val_ratio, seed
- `model`: clip_model_name (must be "ViT-B/32"), num_classes (500), freeze_clip, feature_dim
- `train`: batch_size, epochs, lr, weight_decay, warmup_epochs, amp, scheduler
- `eval`: batch_size
- `output`: log_dir, submission_dir
